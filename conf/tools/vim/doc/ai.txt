==============================================================================
AI.VIM DOCUMENTATION                                         ai_doc.txt
==============================================================================

The ai.vim library is a heavily text-driven, marker-based LLM integration 
for Vim. It reads specific "marker" lines in your buffer to determine what 
to send to the model, whether to include file/URL contexts, and whether to 
treat the interaction as a standalone query or a threaded conversation.

KEY BINDING
-----------
Inside any `.ai` file, use `<Ctrl + g>` (in normal or insert mode) to 
trigger the AI to process the current block.

1. BASIC QUERIES & STARTING THREADS (>>> input)
-----------------------------------------------
Use the `>>> input` marker to ask a standalone question or start a new 
conversation thread. You can optionally specify the model and a context 
name right after the marker using `model=` and `context=`.

    >>> input model=claude-3
    Write a python function to reverse a string.

Press `<C-g>`. The plugin will fetch the text, send it to the model, and 
automatically insert the response below a `>>> output` marker.

2. CONVERSATIONS (>>> continue)
-------------------------------
To continue an existing conversation (so the model remembers previous turns), 
the plugin uses the `>>> continue` marker. 

When you generate a response from an `>>> input`, the plugin will automatically 
append a `>>> continue` marker for you to easily ask follow-up questions.

    >>> input model=gpt-4o
    What is the capital of France?

    >>> output by gpt-4o
    The capital of France is Paris.

    >>> continue model=gpt-4o
    And what is its population?

Running `<C-g>` on the `continue` block sends the entire conversation history 
(starting from the root `>>> input`) so the model has the full context. If you 
want to start a brand new conversation, just type `>>> input` again.

3. PROVIDING CONTEXT (>>> context)
----------------------------------
You can feed files, entire directories, URLs, or system instructions into 
the model's context. Contexts are now *named*, allowing you to define multiple 
contexts in a single file and reference them in your inputs. Context attributes 
must be prefixed with an `@`.

    >>> context my_project
    @role: You are an expert code reviewer.
    @file: src/main.py
    @dir: src/utils/
    @url: https://docs.python.org/3/library/json.html
    @ignore: *.pyc

    >>> input context=my_project model=gpt-4o
    Review the code provided in the context and suggest improvements.

Available context attributes:
  @role:   Sets the system prompt/behavior.
  @file:   Reads a local file (supports images if the model has vision).
  @dir:    Reads all files in a directory.
  @url:    Fetches the raw HTML/text of a webpage to pass to the model.
  @ignore: Adds a glob pattern to skip certain files (useful with `@dir:`).

4. SCOPE MANAGEMENT
-------------------
Because ai.vim scopes interactions between markers, you can isolate multiple 
different conversations in the same file without needing explicit end markers.

  >>> input : Always starts a brand new conversation thread. The active scope 
              for a thread continues downwards until the end of the file or 
              until the very next `>>> input` marker is encountered.

5. CONFIGURATION VARIABLES
--------------------------
Set these globally in your vimrc or init.vim:
  g:ai_models           : Dictionary mapping aliases to full model names.
  g:ai_ignore           : Global list of ignore patterns (e.g. node_modules).
  g:ai_max_file_bytes   : Limits how much text is read from large files.
  g:ai_max_file_lines   : Limits the number of lines read from large files.
